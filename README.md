# 🛒 Zepto E-commerce Inventory Data Analysis using SQL

## 📌 Project Overview

This project demonstrates end-to-end SQL data analysis on a real-world **Zepto e-commerce inventory dataset**. The objective was to simulate the workflow of a Data Analyst by performing data exploration, cleaning, and business analysis using PostgreSQL.

The project focuses on transforming raw inventory data into meaningful business insights that can support pricing strategies, inventory management, and product performance analysis.

---

# 🎯 Objectives

* Build a structured SQL database from raw inventory data.
* Perform Exploratory Data Analysis (EDA).
* Clean and transform inconsistent data.
* Analyze inventory, pricing, discounts, and stock availability.
* Generate business insights using SQL queries.

---

# 🛠 Tech Stack

* **Database:** PostgreSQL
* **Language:** SQL
* **Tools:** pgAdmin 4
* **Dataset:** Zepto E-commerce Inventory Dataset (Kaggle)

---

# 📂 Dataset Information

The dataset contains product inventory information scraped from Zepto's online grocery platform.

### Features

| Column                 | Description                  |
| ---------------------- | ---------------------------- |
| sku_id                 | Unique Product ID            |
| name                   | Product Name                 |
| category               | Product Category             |
| mrp                    | Maximum Retail Price         |
| discountPercent        | Discount Offered (%)         |
| discountedSellingPrice | Selling Price After Discount |
| availableQuantity      | Available Inventory          |
| weightInGms            | Product Weight               |
| outOfStock             | Stock Availability           |
| quantity               | Units Per Pack               |

# 📋 Project Workflow

## 1️⃣ Database Creation

* Created a PostgreSQL database.
* Designed the inventory table with appropriate data types.
* Imported CSV dataset into PostgreSQL.

---

## 2️⃣ Data Exploration (EDA)

Performed exploratory analysis to understand the dataset.

* Total number of products
* Distinct product categories
* Null value detection
* Duplicate product analysis
* In-stock vs Out-of-stock comparison
* Product distribution across categories

---

## 3️⃣ Data Cleaning

Improved data quality by:

* Removing invalid records
* Eliminating zero-priced products
* Converting prices from paise to rupees
* Standardizing pricing values
* Validating inventory information

---

## 4️⃣ Business Analysis

Solved multiple business problems using SQL.

Some of the analyses include:

* Top discounted products
* Highest-priced products
* Out-of-stock premium products
* Category-wise inventory analysis
* Estimated revenue by category
* Average discount by category
* Products offering the best value per gram
* Inventory weight distribution
* Product segmentation based on weight
